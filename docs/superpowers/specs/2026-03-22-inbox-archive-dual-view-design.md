# 收件箱 / 归档 双视图设计

## 背景

当前应用只有一个列表视图，未分类和已归档文章混在一起通过状态筛选区分。用户希望将两者分离：未分类（Triage）保持现有的快速处理流程，归档（Archive）提供以标签为核心的浏览和回顾体验。

## 设计概览

在现有 FilterBar 上方增加 `[收件箱] [归档]` Tab 切换。两个 Tab 共享同一页面，切换时替换下方内容区域。Tab 状态用 React state 管理，切换时各自的筛选状态保留在内存中。

### 收件箱 Tab

保持现有全部功能不变：FilterBar（状态/来源/标签/时间范围/搜索/随机排序）、紧凑列表、阅读面板、键盘快捷键、归档/删除/跳过操作。

数据范围：自动添加 `location != 'archived'` 过滤条件，不再需要用户手动排除。FilterBar 的状态下拉中移除"已归档"选项（归档文章只在归档 Tab 中出现）。

### 归档 Tab

#### 标签云区域

- 顶部展示所有出现在归档文章中的标签，标签大小（font-size）反映该标签下的文章数量
- 缩放规则：font-size 在 12px–24px 之间线性插值，最小计数映射 12px，最大计数映射 24px；所有标签计数相同时统一 16px
- 点击标签 toggle 选中/取消，支持多选
- 多选时使用交集模式：只显示同时包含所有选中标签的文章
- 选中标签时在标签云下方显示"已选"行，每个标签带 × 可单独移除，末尾有"清除"按钮
- 标签云旁边保留搜索框

#### 分组列表

- 按 tier-1 标签（来自 `tag-vocab.ts` 的 `TIER1_DOMAIN`）分组，前端完成分组逻辑
- 每组显示组头：标签名 + 本组文章数（去重后），可折叠/展开，默认全部展开
- 组内为紧凑列表，每行显示：来源类型 badge、标题、评分星标、日期
- 一篇文章有多个 tier-1 标签时，在每个匹配的组里都出现
- 没有 tier-1 标签的文章归入"未分类"组
- 标签云筛选后，只显示包含匹配文章的分组
- 列表顶部显示去重后的文章总数（与分组计数之和区分）

#### 阅读面板

- 与现有阅读面板类似的布局
- 保留：标签编辑器（TagEditor）、评分（SignalRating）、翻译按钮、AI 标签建议
- 隐藏：归档按钮、删除按钮、跳过按钮、来源类型选择器（已归档文章不需要这些操作）
- 键盘快捷键：禁用 Enter（归档）、D（删除）、S（跳过）；保留 ↑↓（导航）、1-5（评分）、E（编辑标签）、T（AI 标签建议）
- 编辑标签或评分后实时保存
- 编辑标签后，如果 tier-1 标签发生变化，分组列表实时更新（文章从旧组消失/在新组出现）

## 数据流

### 归档标签云数据

新增 API `GET /api/archive/tags` 返回归档文章的标签统计：

```typescript
// 返回值
{ tags: Array<{ tag: string; count: number }> }
```

查询条件：`location = 'archived'`，统计每个标签出现的文章数。

### 归档列表数据

新增 API `GET /api/archive/excerpts` 参数：

- `tags`: 逗号分隔的标签列表（交集筛选）
- `search`: 搜索关键词
- `limit`: 分页大小，默认 200
- `offset`: 分页偏移

返回扁平列表（分组在前端完成，使用 `tag-vocab.ts` 的 `TIER1_DOMAIN` 常量）：

```typescript
{
  items: ExcerptItem[];  // 轻量类型，同收件箱列表（id, title, source_type, source_name, signal, status, published_at, tags）
  total: number;
}
```

前端根据每篇文章的 tags 与 `TIER1_DOMAIN` 的交集进行分组。

### 归档文章编辑

复用现有 `PATCH /api/excerpts/[id]` 接口。编辑限制在客户端实施：归档模式下 UI 仅暴露 `tags` 和 `signal` 字段的编辑控件。

## 组件结构

```
page.tsx
├── ViewTabs (新增: 收件箱/归档切换)
├── [收件箱模式]
│   ├── FilterBar (现有，移除"已归档"状态选项)
│   ├── ExcerptList (现有)
│   └── ReadingPanel (现有)
└── [归档模式]
    ├── ArchiveFilterBar (新增: 标签云 + 已选标签 + 搜索)
    ├── ArchiveGroupList (新增: 前端分组 + 折叠列表)
    └── ReadingPanel (复用，传入 archiveMode=true)
```

### 新增组件

| 组件 | 职责 |
|---|---|
| `ViewTabs` | 顶部 Tab 栏，管理 activeView 状态 |
| `ArchiveFilterBar` | 标签云渲染（font-size 缩放）、多选交集筛选、已选标签展示、搜索 |
| `ArchiveGroupList` | 接收扁平列表 + TIER1_DOMAIN，前端分组、折叠/展开、组内紧凑列表 |

### 修改组件

| 组件 | 修改内容 |
|---|---|
| `page.tsx` | 增加 activeView 状态，条件渲染收件箱/归档内容，两个 Tab 各自维护独立的筛选/选中状态 |
| `FilterBar` | 状态下拉移除"已归档"选项 |
| `ReadingPanel` | 新增 `archiveMode` prop：隐藏归档/删除/跳过按钮和来源类型选择器，禁用 Enter/D/S 快捷键 |

## 空状态和加载

- 归档为空时：显示"还没有归档文章"提示
- 标签筛选无结果时：显示"没有同时包含这些标签的文章"，带"清除筛选"按钮
- 数据加载中：显示加载指示器（与收件箱一致）

## 交互细节

- Tab 切换时保留各自的筛选状态（React state），切回来时恢复
- 归档分组默认全部展开，手动折叠的状态在 Tab 切换后保留
- 点击归档列表中的文章时，右侧阅读面板显示内容
- 标签云中 tier-1 标签和非 tier-1 标签都展示，但分组只按 tier-1 标签
